# 难点
+ 对于事件创建，可使用 JSON 对象存储事件编号 id，事件状态 status 和事件内容 title，再使用 localStorage 存入本地浏览器，无论关闭或者刷新，都不会丢失。
+ 在创建事件时对其标签设置自定义属性 data-id 编号，便可通过获取 dataset.id 从而获取本地事件键值对，很方便地修改或删除事件。
+ 通过存储 JSON 对象，可根据其 status 值筛选待办和已办事件，分别渲染至待办事件展示栏和回收站。
+ 事件委托，通过将事件绑定至父元素，借助冒泡特性，实现对渲染页面后生成的 div 元素进行事件绑定。
+ 对于同一个元素的事件绑定，可通过条件语句识别内容，从而实现不同的功能，避免对同一个元素绑定两次相同事件可能引发的问题。
